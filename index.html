Hexo 架設部落格筆記

為什麼要使用 Hexo 架設部落格

  • 低成本：使用 GitHub Pages 免費架設部落格
  • 高擴充性：Hexo 有許多主題可以選擇，也可以客製化自己的主題
  • 高自由度：可以自由地選擇和修改部落格的樣式和功能
  • 高效率：Hexo 有許多套件可以選擇，也可以客製化自己的套件

另外過程中可以學習使用 Git 和 GitHub 來管理部落格的版本，也可以學習到如何使用 Markdown 來撰寫部落格文章。

安裝必要套件

基本的安裝步驟參考官方的 documentation 就可以完成。

常用指令

  • hexo new "貼文名稱" ：建立新文章
  • hexo server ：啟動伺服器
  • hexo generate ：產生靜態檔案
  • hexo deploy ：部署到遠端伺服器 (安裝 hexo git 套件之後，在 _config.yml 設定 deploy 的相關資訊)
  • hexo clean ：清除快取，當出現奇怪問題的時候可以用一下。在使用 hexo generate 和 hexo deploy 之前，先執行 hexo clean 清除快取，可以避免一些不必要的錯誤。

常見錯誤及注意事項

  1. 在 _config.yml 設定的時候要注意，有的冒號後面需要空格
  2. hexo init 之後，還要去下載 theme 的主題，不然本地看得到主題，但是實際上上傳上去之後是沒有的。
  3. 如果你跟我一樣將 Github 資料夾存在電腦的「檔案」資料夾中，iCloud又有設定自動管理檔案，可能舊的檔案會被從本地移除，這個時候可能就要先手動將全部資料夾都下載下來，避免無法順利部署或是生成靜態檔案。
